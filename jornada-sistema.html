Diagnóstico de mercado

Por que não existe uma solução pronta

Foram avaliadas 8 plataformas financeiras e jurídicas: Asaas, Computar, Astrea, MaisJurídico, Projuris, ContaAzul, Omie e Nibo. Algumas cobrem honorários básicos e rateio manual entre advogados — especialmente Computar e Astrea. Nenhuma cobre simultaneamente os três requisitos que definem o fluxo do escritório: reconhecimento de transações por padrão, cruzamento de beneficiários de processos coletivos e integração com ADV Box. A lógica de negócio é construída sob medida porque a combinação é única.

  • Reconhecimento de transações por padrão
    Pattern learning por escritório — na primeira ocorrência pede confirmação; nas seguintes, categoriza automaticamente. O sistema aprende com o uso do E&P, não de outro escritório.
  • Cruzamento de beneficiários de processos coletivos
    FUNDEF, IPCEP, Bom Jardim e Sirinhaém — beneficiários pagos em lotes precisam ser cruzados com o extrato por nome, CPF e número de processo. Não existe ferramenta no mercado que faça essa ponte.
  • Regras de honorários configuráveis por sindicato
    Percentuais diferentes por sindicato (10%, 20%, 30%) e por advogado responsável. Ferramentas prontas operam com tabelas fixas que não refletem a estrutura do escritório.
  • Integração com ADV Box
    Cruzamento direto entre transações bancárias e os 572 processos jurídicos ativos no ADV Box. Nenhum SaaS financeiro ou jurídico disponível no mercado faz essa integração.
Abordagem: construído em Python, aproveitando componentes prontos onde possível — sem reescrever o que já existe no mercado.

Segurança

Como os dados do escritório são protegidos

O sistema foi desenhado para que nenhuma informação sensível do escritório fique exposta — nem credenciais bancárias, nem dados de clientes, nem chaves de acesso.

🔐
Login com Google — sem senha nova
O acesso é feito com o e-mail do Google Workspace do escritório. O Google é responsável pela autenticação. Se quiserem, ativam verificação em dois fatores pelo próprio Google.
🔒
Comunicação criptografada — HTTPS
Toda comunicação entre o sistema e os dispositivos é criptografada. Ninguém consegue interceptar os dados em trânsito.
💾
Dados armazenados com criptografia
Todos os lançamentos, processos e beneficiários ficam criptografados no banco de dados. Mesmo que alguém acesse o servidor fisicamente, não consegue ler nada.
🏦
Senha do banco nunca entra no sistema
Waldir exporta o extrato do banco e faz o upload. As credenciais bancárias ficam exclusivamente no app do banco — nunca tocam o sistema. Na Fase 2 (Open Finance), a autorização é feita pelo próprio app do banco.
🔑
Chaves de integração protegidas
As chaves de acesso às ferramentas integradas (IA, ADV Box, Gmail) ficam guardadas no Google Secret Manager — nunca no código, nunca expostas.
📝
Auditoria — tudo registrado
Toda ação no sistema fica registrada: quem confirmou qual lançamento, quando, de qual dispositivo. Rastreabilidade completa.
🗄️
Backup automático diário
O banco de dados tem backup automático todo dia. Os arquivos ficam também no Google Drive. Os dados do escritório nunca se perdem.
🇧🇷
Dados no Brasil
O servidor e o banco de dados ficam hospedados no Brasil, dentro da infraestrutura Google. Os dados não saem do país.

Como o sistema pensa

Fluxo de identificação de cada transação

Quando um pagamento entra no extrato, o sistema percorre esta sequência até identificar a origem.

Já foi visto antes?
Pattern matching
Verifica se aquele CNPJ, nome ou padrão de valor já foi categorizado antes. Se sim: confirma automaticamente e aplica a mesma regra. Sem interação de Waldir.
Mensalidade de sindicato?
Cadastro fixo
Cruza o CNPJ ou nome do pagador com o cadastro dos 15 sindicatos. Se encontrar: registra automaticamente com o percentual de repasse correspondente e marca como pago.
PIX de pessoa física?
Correspondência de nomes
O banco às vezes trunca o nome. O sistema compara com a base de beneficiários e calcula a semelhança. Acima de 90%: aceita automaticamente. Entre 70–90%: exibe sugestão para Waldir confirmar. Abaixo de 70%: pede classificação manual.
Depósito judicial — BB?
Portal público
O extrato do BB traz um número de protocolo. O sistema consulta o portal público do Banco do Brasil com esse número + CNPJ do escritório. Retorna automaticamente: processo, sindicato, valor, data, juros. Waldir não faz nada.
Depósito / TEV — Caixa?
E-mail automático
Sem portal público disponível. O sistema envia automaticamente um e-mail para Tiago com o valor e a data. Quando Tiago responde, a IA lê o documento e registra o processo identificado sem ação de Waldir.
Não identificado
Classificação manual
Waldir classifica manualmente uma única vez. O sistema aprende e reconhece automaticamente nas próximas vezes. Com o uso, o número de lançamentos manuais vai diminuindo.
